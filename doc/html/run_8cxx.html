Detailed Description

Code to run the analysis.

Author:
daid
Version:
1.0

Reads one or more input ROOT Trees generated by ridf2root.C
Calls Analyzer.cxx on the data, and writes out histograms.
Allows user input or automation to select run files.
Reads input/output directories and run header informtion from run.conf
Runtime options dictate how data will be analyzed (raw and/or detailed) and which type of data will be analyzed (ssd pad, ssd strip, ppac, tpc).

Fortran energy loss subroutine.

Uses Ziegler's method for energy loss. A subroutine for enewz.

Parameters:
[in]z1Proton number of ion.
[in]m1Nucleon number of ion.
[in]eEnergy of ion (MeV/A).
[in]matter1Matter the ion passes through, as defined in SNKE_MATTER.INC — Case-insensitive.
Must be 32 characters followed by the \0 termination character.
[in]unit_pressureDefine the units of pressure: 0=solid target ; 1=Torr ; 2=mbar ; 3=atm.
[in]pressureDefine the value of the pressure (0 for solid target).
[in]temperatureDefine the temperature in Kelvin (0 for solid target).
[in]unit_thickDefine the units of thickness: 1=mm ; 2=mg/cm2
[in]thick1Define the value of the thickness.
[out]aft_eneIon energy after energy loss in matter (MeV/A).

main function of run.cxx which controls everything else.

The main function gets the runtime options, can interact with the user to specify run number(s), reads the input ROOT Tree, passes the Tree to Analyzer, Writes the histograms to a new Tree, and optionally launches a ROOT session with a TBrowser attached to the output .root file.
Essentially everything is controlled from here.
The parameters are just the standard form for getopt parsing.
